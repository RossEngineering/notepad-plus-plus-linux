# v1.0.0 Release Checklist (Draft)

Draft created on 2026-02-13.
Use this checklist to promote from satisfactory RC3 (`v0.9.9-rc.3`) to `v1.0.0`.

## Final scope and freeze

- [ ] RC3 accepted as release baseline.
- [ ] Feature freeze confirmed (only critical fixes/docs/release operations after freeze).
- [ ] Final blocker review completed with zero unresolved P0 issues.

## Quality and reliability gate

- [ ] Linux required CI lanes green for 7 consecutive days.
- [ ] Sanitizer lane green for release candidate window.
- [ ] Full regression suite pass (editor core, language workflows, skinning, extensions, crash recovery).
- [ ] No unresolved data-loss regressions.

## Compatibility and distro gate

- [ ] Arch derivatives validation evidence attached.
- [ ] Ubuntu LTS validation evidence attached.
- [ ] Fedora stable validation evidence attached.
- [ ] Any remaining distro issues documented with mitigation and owner.

## Packaging and supply chain gate

- [ ] Release artifacts generated (`.tar.xz`, `.sha256`, optional `.asc`).
- [ ] Checksums verified independently from generated files.
- [ ] Artifact naming/version strings match `v1.0.0`.
- [ ] Reproducible release verification completed (where applicable).

## Documentation and migration gate

- [ ] `docs/releases/v1.0.0.md` finalized.
- [ ] Migration notes finalized for users moving from beta/rc builds.
- [ ] Compatibility and support statements updated in `README.md` and related docs.
- [ ] Known limitations list reviewed and accepted for GA.

## Publish

- [ ] `master` pushed with final release state.
- [ ] Tag `v1.0.0` created and pushed.
- [ ] GitHub Release published with matching artifacts.
- [ ] Post-release validation smoke check completed from release artifacts.
