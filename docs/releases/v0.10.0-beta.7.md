## Scope for this prerelease

1. Fix Arch package metadata compatibility for package-manager install paths.
2. Publish corrected maintenance prerelease after `v0.10.0-beta.6` Arch install failure report.

## Highlights

1. Arch package archive creation now writes top-level metadata entries without `./` prefixes:
   - `.PKGINFO`
   - `.INSTALL` (when present)
2. Pacman metadata parsing compatibility restored:
   - `pacman -Qip <package.pkg.tar.zst>` now succeeds
   - GUI package installers that rely on libalpm metadata parsing can prepare transactions correctly
3. Existing package outputs remain unchanged:
   - Arch `.pkg.tar.zst`
   - Debian/Ubuntu `.deb`
   - Fedora `.rpm`
   - release tarball + checksum

## Known limitations

1. Marketplace remains local-index only; remote catalog/discovery is not implemented in v1.
2. Extension resource/startup metrics are best-effort heuristics, not kernel-level telemetry.
