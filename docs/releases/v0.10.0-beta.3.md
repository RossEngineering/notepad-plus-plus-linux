## Highlights

1. Native Linux package artifacts in release automation:
   - Arch (`.pkg.tar.zst`)
   - Debian/Ubuntu (`.deb`)
   - Fedora (`.rpm`)
2. Desktop integration polish:
   - `.desktop` actions for new file and open recent
   - category cleanup and stricter desktop integration validation
3. Local installer/uninstaller improvements:
   - launcher/file-handler registration hardening
   - update-channel override support (`--update-channel stable|candidate`)
4. Update channel model:
   - packaged default channel marker (`default-update-channel`)
   - Preferences setting (`Stable` / `Candidate`)
   - Help -> Check for Updates resolves channel-specific release endpoint
5. Wayland QA:
   - dedicated headless Wayland smoke script (`weston` + `QT_QPA_PLATFORM=wayland`)
   - dedicated CI workflow lane (`Linux Wayland QA`)

## Known limitations

1. Update channel currently selects release feed endpoint only; in-app download/apply update flow is still out of scope.
2. Wayland QA lane is smoke coverage, not a full compositor/desktop-environment behavior matrix.
